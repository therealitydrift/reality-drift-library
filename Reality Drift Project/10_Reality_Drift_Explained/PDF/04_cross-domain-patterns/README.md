# Cross-Domain Patterns

This folder contains shared structural patterns that appear across domains such as education, healthcare, media, and organizations.

These patterns describe how systems translate reality into measurable, comparable, and scalable forms—and how those transformations introduce loss, distortion, and drift.

## Core Concept

Across domains, the same structure emerges:

Reality → Measurement → Metrics → Optimization → Representation → Narrative

Each layer enables scale, but also introduces distance from what is being represented.

## What This Includes

- Representation Stack variants across domains  
- Shared mechanisms such as:
  - proxy optimization  
  - semantic compression  
  - abstraction layering  
  - feedback degradation  
- Cross-domain explanations of drift and misalignment  

## Why This Matters

These patterns are not domain-specific. They are system-level behaviors.

A system can continue to function, improve its metrics, and produce coherent outputs while gradually losing alignment with reality.

## Key Insight

> The map keeps updating even after the territory disappears.
